# Modern UI Redesign

**Date:** 2026-03-08
**Status:** Design approved, ready for specification

## Problem

The current UI is functional and clean but visually dated — manually styled Tailwind components with no design system, flat top-nav layout, and no visual hierarchy between advisor and client audiences.

## Approach

Sidebar + shadcn/ui Refresh — swap top navbar for a collapsible dark sidebar, adopt shadcn/ui components built on the existing Radix UI + Tailwind + CVA stack, and differentiate advisor vs client visual modes.

---

## Design Sections

### 1. Navigation & Layout

- Replace top navbar with a collapsible left sidebar
  - Expanded: 240px — icon + label
  - Collapsed: 64px — icon only, tooltip on hover
- Page shell: `sidebar | main content` (two-column)
- Main content retains `max-w-7xl` centering
- DataFreshness bar moves to sidebar footer (always visible)
- ClientViewToggle moves to a toolbar strip at top of main content (Dashboard route only)
- `/admin` System Health entry in sidebar with separator line and dimmer visual treatment

**Sidebar nav order:**
```
[logo]
─────────────
Dashboard
Goal Planner
Risk Profiler
Scenarios
─────────────
System Health   ← staff-only, dimmer
─────────────
[avatar] Advisor name
Sign out
```

### 2. Typography & Color Tokens

**Font:** Inter via `@fontsource/inter` (no Google Fonts dependency)

**Type scale:**
| Role | Token |
|---|---|
| Page titles | `text-xl font-semibold` |
| Section headers | `text-base font-semibold text-gray-900` |
| Body / table | `text-sm text-gray-700` |
| Captions / meta | `text-xs text-gray-500` |

**Color palette:**
| Token | Value |
|---|---|
| Page background | `gray-50` |
| Sidebar background | `gray-900` (dark) |
| Sidebar text | `gray-300` / active `white` |
| Card background | `white` + `shadow-sm` |
| Primary blue | `blue-600` |
| Border | `gray-200` |

Dark sidebar against light content — premium contrast without full dark mode.

**Spacing:** `gap-6` rhythm for page sections, `p-5` standard card padding.

---

### 3. Component System (shadcn/ui)

shadcn/ui CLI copies component source into the project (no runtime dependency). Drops cleanly onto existing Radix UI + Tailwind + CVA.

**Components to adopt:**
| Component | Replaces | Used in |
|---|---|---|
| `Button` | Manual class strings | Every form, JobCard, FilterBar |
| `Badge` | Status span classes | JobCard, AssetTable risk levels |
| `Card` / `CardHeader` / `CardContent` | `bg-white rounded-lg border p-4` | JobCard, ScoreBreakdown, GoalForm |
| `Select` | Raw Radix Select wiring | FilterBar, RiskProfiler |
| `Input` / `Label` | Native inputs + manual Tailwind | LoginForm, GoalForm |
| `Skeleton` | "Loading..." text / blank states | Tables, cards, charts |
| `Tooltip` | Raw Radix Tooltip wiring | AssetTable column headers |
| `Separator` | `<hr>` / border dividers | Sidebar sections |

**Stays custom:** ScoreMeter (SVG gauge), Recharts components, DataFreshness bar.

**Migration strategy:** Sidebar shell built first with shadcn/ui. Existing components migrate opportunistically — no big-bang rewrite.

---

### 4. Advisor vs Client View Differentiation

Presentation layer changes only — same data, different visual treatment. No new routes.

**Advisor view (default):**
- Full sidebar expanded, all nav items accessible
- AssetTable: all columns (metrics, scores, liquidity, tax)
- Compact row height, dense data display
- DataFreshness in sidebar footer
- System Health accessible

**Client view (toggled):**
- Sidebar auto-collapses to icon-only (less chrome)
- AssetTable: simplified — name, category, 1Y/3Y returns, risk level only
- ProductPins becomes the hero section — large cards layout
- WhatsApp share button prominent
- Slightly larger typography (`text-base` body)
- DataFreshness hidden, System Health hidden

**ClientViewToggle:** Polished segmented control:
```
[ Advisor View ]  [ Client View ]
```

---

### 5. Micro-interactions & Loading States

**Skeleton loaders:**
- AssetTable: 5 skeleton rows on first load
- Admin JobCard grid: 6 skeleton cards on initial load
- GoalForm: skeleton for chart area while computing

**Transitions (Tailwind only — no Framer Motion):**
- Sidebar expand/collapse: `transition-all duration-200`
- Route change: subtle fade-in (CSS opacity)
- ClientViewToggle: sliding pill indicator
- JobCard Run Now: inline spinner while triggering

**Hover states:**
- Table rows: `hover:bg-blue-50/40` (subtle blue tint)
- Sidebar nav active item: `border-l-2 border-blue-500` left accent bar
- JobCards: `hover:shadow-md transition-shadow`

## Out of Scope (Parking Lot)

- Full dark mode (deferred — sidebar is dark, main content stays light)
- Storybook component documentation
- E2E tests for new components
- Separate client portal route/subdomain

## Success Criteria

- Sidebar navigation feels natural within 1 session for existing users
- Client view hides all internal tooling (no advisor-only columns/links visible)
- No visible layout shift during skeleton → content transition
- All existing functionality works identically after migration
- Bundle size increase < 50KB gzipped from shadcn/ui additions
